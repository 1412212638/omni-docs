# 计费与退款说明

生效日期：2026 年 3 月 25 日

本页说明 OmniRouters 当前采用的预充值计费模式以及退款处理原则。本页构成《[用户协议](/legal/terms)》的补充说明。

## 1. 计费模式

OmniRouters 当前主要采用预充值余额模式：

- 您可向账户充值，用于后续 API 调用、模型使用和相关资源消耗
- 平台会根据实际调用量、模型路由、资源使用情况和当时适用的费率从余额中扣费
- 账户余额不构成银行存款、储值支付工具或任何形式的信托资金，也不计付利息

## 2. 费用显示与对账

您通常可以在控制台、账单记录或相关页面查看：

- 充值记录
- 扣费记录
- 当前余额
- 退款状态（如有）

如您发现异常扣费、重复计费或与预期明显不符的账单情况，请尽快联系我们并提供相关记录。

## 3. 退款规则

### 3.1 通常不可退款的情形

- 已实际消耗的接口调用费用、模型费用和资源费用
- 已使用、已发放或已失效的测试额度、赠送余额、促销余额或活动权益
- 因您违反《[用户协议](/legal/terms)》、滥用服务、欺诈、违法活动或其他应由您负责的原因导致账户被限制、暂停、终止或封禁后的剩余余额

### 3.2 可以申请退款的情形

- 尚未消耗的预充值余额
- 平台重复扣费、错误计费或明显系统异常导致的多收费用
- 未授权支付、支付渠道重复入账或服务未成功开通的情况
- 因 OmniRouters 一方原因导致相关服务长期无法正常使用的情形

### 3.3 手续费扣除

对于可退款的未消耗预充值余额，退款金额通常会扣除实际发生的：

- 支付通道手续费
- 银行手续费
- 汇兑费用
- 其他必要处理成本（如有）

如退款属于 OmniRouters 一方错误引起的重复扣费、错误计费或未授权支付等情形，我们将在核实后按情况安排全额退款或合理部分退款。

## 4. 如何申请退款

请通过以下方式提交退款申请：

- 技术支持：`support@omnirouters.com`

建议在申请中提供：

- 账户邮箱或账户名称
- 充值订单号、付款时间和付款方式
- 拟申请退款的金额及原因
- 付款凭证、账单截图或其他必要证明

为防止欺诈和误退，我们可能要求您完成身份验证或补充材料。

## 5. 退款处理与到账

- 我们会在收到完整材料后于合理期间内完成审核
- 审核通过后，我们通常会按原支付路径退回；如原路径不可用，则可能与您协商其他合法可行方式
- 实际到账时间取决于支付机构、银行、卡组织或相关第三方处理时效

## 6. 计费争议

如您对账单、余额或扣费结果有异议，请在发现问题后尽快与我们联系。我们可能要求您提供日志、订单、模型配置或支付凭证，以完成核查。

## 7. 费率与政策更新

我们可根据产品、模型、支付渠道和业务安排更新费率、充值规则或退款政策。更新版本将在网站、文档或控制台中发布，并自标注日期起生效。
