# 子处理者与第三方服务类别

生效日期：2026 年 3 月 25 日

OmniRouters 可能在为您提供服务的过程中使用第三方服务商、合作伙伴和子处理者，以完成托管、支付、客户支持、安全、日志、模型调用和其他运营活动。本页说明我们公开披露的第三方服务类别和管理原则。

## 1. 披露范围

本页主要说明 OmniRouters 可能使用的子处理者和第三方服务类别，并不构成对任何特定厂商持续不变的承诺。具体供应商可能会因产品架构、区域、支付方式、模型供应情况和合规要求而调整。

如企业客户根据合同、采购流程或合规要求需要获取更细化的当前列表，可与我们联系。

## 2. 常见类别

| 类别 | 典型用途 | 可能涉及的数据 |
| --- | --- | --- |
| 云基础设施与存储 | 计算、网络、对象存储、备份、日志承载 | 账户数据、日志、配置、业务数据 |
| 支付与风控服务 | 充值、退款、支付确认、反欺诈 | 订单信息、支付状态、交易参考号 |
| 客服与沟通工具 | 工单、邮件、客户支持、通知 | 联系方式、工单内容、附件 |
| 安全与运维工具 | 监控、告警、审计、故障分析、访问控制 | 日志、设备信息、事件记录 |
| 第三方模型与推理基础设施 | 响应模型调用和推理请求 | 提示词、输入、上下文、文件、输出 |
| 专业服务机构 | 审计、法律、合规、财务支持 | 限于为履行相关职责所必需的信息 |

## 3. 我们如何管理子处理者

我们通常通过以下方式管理相关第三方：

- 基于业务需要、技术能力、安全性和合规性进行准入评估
- 在适用范围内签署保密、数据处理、服务条款或其他合同性安排
- 要求其仅在提供服务所必需的范围内处理数据
- 对跨境传输、访问控制、事件通知、保留与删除机制提出合理要求
- 根据服务变化、风险变化或监管要求对供应商进行复审

## 4. 跨境与模型路由说明

由于 OmniRouters 支持不同模型与基础设施服务，相关数据可能根据您所选的模型、区域、支付渠道或支持方式转移至香港特别行政区以外的地区。详情请同时参阅《[隐私政策](/legal/privacy)》和《[AI 使用说明](/legal/ai-usage)》。

## 5. 企业客户与合同安排

对于需要更详细数据处理披露、子处理者通知机制或转移保障条款的企业客户，我们通常可结合具体服务形态、合同范围和适用法律要求提供相应说明。相关安排通常会在订单、主服务协议、数据处理附录或其他商务文件中明确。
